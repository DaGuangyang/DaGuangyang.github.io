Go 切片

切片

切片Slice是一个相同元素的可变长度的序列，是基于数组类型做的一层封装。
切片是一个引用类型，它的内部结构包括地址,长度,容量。

基于数组定义切片

由于切片的底层就是数组，所以可以基于数组定义切片。基于数组 Array[x,y]定义的切片的长度就是切片中的元素数，但是容量与[x,y]中的x有关，与y无关。

切片再切片

除了基于数组定义切片，还可以基于切片来定义切片。

基于切片定义切片时，需要注意不能超过原数组的长度，否侧将会出现数组越界,但是可以超过基于的切片的长度。

使用make()函数构造切片

若将容量省略，则容量默认与长度相同，make()函数构造的切片默认分配了内存空间，元素值为元素类型的零值。

切片的本质

切片的本质就是对底层数组的封装，它包含了三个信息：底层数组的指针、切片的长度、切片的容量。
举例：
a := [8]int{0, 1, 2, 3, 4, 5, 6, 7}


切片的容量是底层数组从切片的第一个元素到最后一个元素的数据量。

切片不能直接比较

切片之间是不能直接比较的，不能使用==来判定两个切片是否含有全部相等元素。切片唯一合法的比较操作是和nil进行比较，一个nil值的切片没有底层数组，且长度和容量都是0，但是不能说一个容量和长度都是0的切片就是nil。

正确的方式应该是创建一个相同元素类型的新切片，然后将原切片的元素遍历到新切片中。

可以通过查看$GOROOT/src/runtime/slice.go源码，其中扩容相关代码如下：

可以发现四种情况：

  1. 新申请容量大于2倍的旧容量，则最终容量为新申请的容量。
  2. 新申请容量小于2倍的旧容量，且旧切片的长度小于1024，则最终容量为旧容量的2倍。
  3. 新申请容量小于2倍的旧容量，且旧切片的长度大于1024，则旧容量循环增加自身的四分之一，直到这个数值大于等于申请的容量，这个数值就是最终的容量。
  4. 最终容量计算值溢出，最终容量就是新申请容量

使用copy()函数复制切片

因为切片是引用类型，所以a和b其实都是指向了同一块内存地址。修改b的同时a也会发生变化。
Go语言中的内建函数copy()可以将一个切片的数据复制到另外一个切片空间中。

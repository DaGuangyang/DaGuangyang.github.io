Go 结构体

类型别名和自定义类型

自定义类型

Go语言中可以使用type来进行自定义类型。

通过type自定义的类型newType是一种新的类型，但它具有int的特性。

类型别名

类型别名是Go1.9版本添加的新的功能。

NEWNAME与TYPE本质上是同一个类型，例如基本数据类型中的byte 与 uint8、rune 与 int32

自定义类型与类型别名之间的区别

main.newType1的意思是main包下的newType1类型。

结构体

Go语言使用struct来实现面向对象。

结构体的定义

使用type和struct来定义结构体。

类型名：表示结构体的名称，在同一个包内该名称不能重复。
字段名：表示结构体字段名，结构体中字段名是唯一的。
类型：表示结构体字段类型

方法和接受者

Go语言中的方法（method）是作用与特定类型变量的函数吗。这种特定类型叫做接受者。

接受者变量:接受者变量名官方建议使用接收者类型的小写首字母，例如：接收者类型是people结构体类型，那么接收者变量名应为p。
接收者类型:接收者类型和参数类似，可以是指针类型和非指针类型。
方法名、参数列表、返回参数：具体格式与函数定义相同。

指针类型的接受者

调用方法时修改接收者指针的任意成员变量，在方法结束后，修改都是有效的。

值类型的接受者

当方法作用于值类型接收者时，Go语言会在代码运行时将接收者的值复制一份。在值类型接收者的方法中可以获取接收者的成员值，但修改操作只是针对副本，无法修改接收者变量本身。

什么时候应该使用指针类型接收者

  1. 需要修改接收者中的值。
  2. 接收者是拷贝代价比较大的大对象。
  3. 保证一致性，如果有某个方法使用了指针接收者，那么其他的方法也应该使用指针接收者。

方法的接收类型是任意的

方法的接收类型是任意的，不仅仅是结构体。

注意：非本地类型不能定义方法，也就是说我们不能给别的包的类型定义方法。 people是一个新的自定类型，并且这个类型是在当前包中的，所以可以定义方法。而string这种内置类型是不能直接定义方法的，需要通过这种方式来进行定义方法。

结构体的匿名字段

结构体允许其成员字段在声明时没有字段名而只有类型，这种没有名字的字段就称为匿名字段。

匿名字段默认采用类型名作为字段名，结构体要求字段名称必须唯一，因此一个结构体中同种类型的匿名字段只能有一个。

嵌套结构体

一个结构体中可以嵌套包含另一个结构体或结构体指针。

嵌套匿名结构体

可以看出想要访问匿名结构体中的字段可以有两种方式：

  1. 推荐使用的方式： 通过结构体.匿名结构体.字段名来访问
  2. 直接通过结构体.匿名结构体字段名 来访问。（当访问结构体成员时会先在结构体中查找该字段，找不到再去匿名结构体中查找）

嵌套结构体的字段名冲突

当一个结构体中有多个字段为结构体时，这多个结构体内的字段名可能会有相同的字段名，所以此时防止出现错误，最好使用结构体.子结构体.字段名的方式来进行访问。

结构体的“继承”

结构体的继承是通过在子结构体中嵌套父结构体实现的，子结构体可以继承父结构体的方法。

结构体字段的可见性

结构体中字段大写开头表示可公开访问，小写表示私有（仅在定义当前结构体的包中可访问）。

结构体与JSON序列化

结构体标签（Tag）

Tag在结构体字段的后方进行定义。

注意事项： 为结构体编写Tag时，必须严格遵守键值对的规则。结构体标签的解析代码的容错能力很差，一旦格式写错，编译和运行时都不会提示任何错误，通过反射也无法正确取值。例如不要在key和value之间添加空格。
结构体标签在将结构体转换为JSON时是非常有用的。
| 标签 | 作用 |
| :———————————— | :———————————————————————- |
| A int `json:"-"` | 字段被本包忽略 |
| A int `json:"myName"` | 字段在json里的键为”myName” |
| A int `json:"myName,omitempty"` | 字段在json里的键为”myName”且如果字段为空值将在对象中省略掉 |
| A int `json:",omitempty"` | 字段在json里的键为”A”（默认值），但如果字段为空值会跳过；注意前导的逗号 |

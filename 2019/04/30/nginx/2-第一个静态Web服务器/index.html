图中默认的html网页是提前准备的资源，它是存放在/srv/nginx/html下的index.html

nginx最核心的功能就是提供http服务，之所以有默认的http服务可以使用，是因为nginx的配置文件中默认就有对应的配置，就是conf/nginx.conf配置文件，这个配置文件是nginx的主配置文件。

该配置文件中最常用的配置语法为http部分、server部分、location部分，http中可以配置多个server，server中可以配置多个location。
一个server对应一个http服务，location为url的一部分例如：www.XXXX.com/localtion,通过location将url中的路径和服务器的某个目录建起关联关系。
nginx会提供一个默认的http服务，这个就是nginx.conf中默认的server块，这个默认server监听在80端口，当我们访问一个http网站时，如果没有指明端口，默认会访问80端口，这一点在配置中listen 80进行了配置。
当我们在浏览器上使用IP访问nginx服务器时，会默认访问nginx服务器上的/srv/nginx/html/index.html文件，这是由该配置决定的。

当我们访问http://IP/这个url时，这个url最后的/其实就是配置文件'location /‘中的”/“,这个location块中有两个配置指令分别是root、index。
root：当前location所对应的文档根目录是哪里，root html 表示当前location的文档根目录是html目录
index：当我们直接访问http://IP/这个url时，没有指明想要获取什么，那么nginx就会吧index.html或index.htm返回给我们。

yum安装的nginx

通过yum安装的nginx，对应的二进制文件自动拷贝到了/usr/sbin目录中，此目录默认已经加入到了环境变量中，所以我们可以在任意目录执行nginx命令启动nginx，而相关的配置文件存放在了/etc/nginx中；默认的server块会配置在/etc/nginx/conf.d/default.conf中，而不是在/etc/nginx/nginx.conf中，在/etc/nginx/nginx.conf中可以发现有

include表示将指定文件中搞得内容包含到当前位置。

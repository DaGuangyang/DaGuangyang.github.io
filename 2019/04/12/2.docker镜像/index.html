2.Docker镜像

发表于 2019-04-12 | 更新于 2019-04-13 | 分类于 docker



在使用docker命令时需使用管理员身份运行

Docker镜像介绍

在上一章介绍到镜像是docker的三大组件之一。docker运行容器需要本地存在对应的镜像。如果镜像不存在本地，docker会从镜像仓库下载（默认是Docker Hub中的仓库）。

获取镜像

  1. 查找想要的镜像
    docker search [选项] image_name
    【选项】：
    -s NUM：可以查看NUM星以上的镜像
    –automated:默认false，显示automated build镜像
    –no-trunk：默认false，以不截断方式查找镜像
  2. 获取想要的镜像
    docker pull [选项] image_name[:tags]
    【选项】：
    -a:下载所有该tags的镜像
    tags：若忽略tags则下载最新版本的镜像

因docker默认的仓库为Docker Hub，在国内使用时速度比较慢，以下三种方式更换(registry.docker-cn.com 为Docker中国官方镜像仓库)镜像仓库

  1. 方法一：
    本次拉取有效
    docker pull registry.docker-cn.com/library/ubuntu:12.04
  2. 方法二：
    当前进程有效
    docker --registry-mirror=https://registry.docker-cn.com daemon
  3. 方法三：
    永久性保留,
    Docker 中国官方镜像加速
    或者使用阿里云的镜像仓库

    配置修改完后重启docker服务systemctl restart docker

列出本地镜像

docker images [选项] [镜像仓库[:镜像标记]]
【选项】：
-a：显示本地所有的镜像文件（含中间影响层，默认不展示中间映像层）
–digests：显示镜像文件的摘要信息
–no-trunc：显示完整的镜像信息

REPOSITORY：镜像仓库
TAG：镜像标签，区分版本
IMANE ID：镜像ID（唯一）
CREATED：镜像创建时间
SIZE：镜像大小

创建镜像

创建镜像的方法一般有两种：从镜像库中下载镜像修改；利用本地文件系统自行创建。

修改已有镜像

  1. 先使用下载的镜像启动容器
    docker run -i -t ubuntu /bin/bash

    root@后面的为该容器的ID
  2. 在容器中进行修改,如安装软件,
    apt-get -yqq update
    apt-get -y install apache2
  3. 退出容器
    exit
  4. 提交更新后的副本
    docker commit [选项] 容器ID 镜像仓库名[:tags]
    【选项】
    -m=”commit message”:指定提交的说明信息
    –author=”author”:镜像作者信息

    创建成功后会返回镜像ID信息
    容器ID除了在之前容器中的交互式中查看之外，还可以通过docker ps -a查看

    创建成功后就可以在docker的本地镜像列表中查看到了

通过Dockerfile文件和docker build创建镜像

首先创建一个/opt/static_web目录来保存Dockerfile，这个目录就是我们的构建环境，然后再编辑Dockerfile文件

FOEM：指定使用哪个镜像作为基础
MAINTAINER：指定作者以及作者邮箱地址
RUN：会在当前镜像中运行该命令
EXPOSE：告诉Docker该容器内的应用程序将会使用容器的指定端口
此处先介绍简单的Dockerfile语法，更多的Docker语法后续介绍

然后执行docker build -t="test/ubuntu:1.0" .
-t:指定仓库和名称，test为仓库名，ubuntu为镜像名，:1.0为tags
.：指定Dockerfile的路径
在build进程执行过程中可以看到第一件事就是将Dockerfile内容上传，以为所有的操作都依据Dockerfile进行的，然后Dockerfile中的指令被一条一条的执行，每一步都创建一个新的容器，在容器中执行指令并提交修改。当所有的指令都执行完毕后返回最终的镜像ID。所有的中间步骤产生的容器都会被删除和清理。

从本地文件系统导入镜像

docker import [OPTIONS] file|URL|- [REPOSITORY[:TAG]]
-c:应用docker指令创建镜像
-m：提交时的说明文字
file|URL:导入镜像的路径
REPOSITORY：仓库名
TAG：标签

上传镜像

用户可以通过docker push命令将自己创建的镜像上传到仓库来共享

将自己的docker镜像上传到阿里云的docker hub

登录阿里云，在容器镜像服务的命名空间中创建一个命令空间，然后在镜像仓库中创建镜像仓库，具体的操作步骤可以查看仓库后的管理

导出和导入镜像

导出镜像

导出镜像到本地文件，可以使用docker save命令

导入镜像

使用docker load从导出的本地文件中再导入到本地镜像

这将会导入镜像以及其的元数据信息（包括标签等）

删除本地镜像

如果要移除本地镜像，可以使用docker rmi REPOSITORY:TAG命令。注意docer rm CONTAINER ID是删除容器。
docker rmi registry.cn-shanghai.aliyuncs.com/daguangy/my_images:v1
注意：删除这个镜像之前要删除所有依赖这个镜像的容器

清理所有未打过标签的本地镜像

docker images可以列出本地镜像，其中很可能会包含很多中间状态的未打过标签的镜像，会占据大量的磁盘空间

Docker镜像介绍

在上一章介绍到镜像是docker的三大组件之一。docker运行容器需要本地存在对应的镜像。如果镜像不存在本地，docker会从镜像仓库下载（默认是Docker Hub中的仓库）。

获取镜像

  1. 查找想要的镜像
    docker search [选项] image_name
    【选项】：
    -s NUM：可以查看NUM星以上的镜像
    –automated:默认false，显示automated build镜像
    –no-trunk：默认false，以不截断方式查找镜像
  2. 获取想要的镜像
    docker pull [选项] image_name[:tag]
    【选项】：
    -a:下载所有该tags的镜像
    tag：若忽略tag则下载最新版本的镜像
    因docker默认的仓库为Docker Hub，在国内使用时速度比较慢，以下三种方式更换(registry.docker-cn.com 为Docker中国官方镜像仓库)镜像仓库
  3. 方法一：
    本次拉取有效
    docker pull registry.docker-cn.com/library/ubuntu:12.04
  4. 方法二：
    当前进程有效
    docker --registry-mirror=https://registry.docker-cn.com daemon
  5. 方法三：
    永久性保留,
    Docker 中国官方镜像加速
    或者使用阿里云的镜像仓库

    配置修改完后重启docker服务systemctl restart docker

列出本地镜像

docker images [选项] [镜像仓库[:镜像标记]]
docker image ls [选项] [镜像仓库[:镜像标记]]
【选项】：
-a：显示本地所有的镜像文件（含中间层镜像，，默认不展示中间层镜像）
–digests：显示镜像文件的摘要信息
–no-trunc：显示完整的镜像信息
-f –filter:过滤，例如-f since=mongo：3.2，是查看mongo：3.2之后建立的镜像，查看之前的镜像使用before。
-q:只显示镜像的ID
–format：列出镜像列表指定的列，如：

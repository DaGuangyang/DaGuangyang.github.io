4.linux根目录及文件类型

发表于 2019-03-26 | 更新于 2019-04-08 | 分类于 linux


linux根目录结构、linux文件类型

linux根目录结构

FHS：文件系统层次化标准 具体可查看FHS标准

  • /bin：所有用户可用的基本命令程序文件

  • /sbin：供系统管理员使用的工具程序

  • /boot：引导加载器必须用到的各静态文件；kernel、initramfs（initrd）、grub等

  • /dev：存储特殊文件或设备文件；字符设备（线性设备：有时序性；键盘、显示器）、块设备（随机设备：无时序性；硬盘）

  • /etc：系统程序的配置文件，只能为二进制，不能是可执行文件

  • /home：普通用户家目录

  • /root：管理员家目录

  • /lib：为系统启动或根文件系统上的应用程序（bin，sbin）提供共享库，以及为内核提供内核模块

    libc.so.*：动态链接的C库

    ld*：运行时链接器/加载器

    modules：用于存储内核模块的目录

  • /bin64：64位系统特有的，存放64位共享库

  • /media：便携式设备挂载点

  • /mnt：其他文件系统的临时挂载点

  • /opt：附加应用程序的安装位置

  • /root：管理员家目录

  • /tmp：临时文件

  • /usr：全局共享的只读文件

    bin，sbin

    lib，lib64

    include：C程序的头文件

    share：命令手册页和自带文档等架构特有的文件

    local：安装本地应用程序，安装用于安装第三方程序、

  • /var：经常放生变化的文件

  • /proc：基于内存（关机不存在），虚拟文件系统，内核及进程存储其相关信息，多为内核参数

  • /sys：基于内存（关机不存在），提供了一种比proc更为理想的访问内核数据的途径

Linux文件类型

  • -：常规文件，即f

  • d：目录文件

  • b：块设备，以“block”为单位进行随机访问

  • c：字符设备，以“character”为单位进行线性访问对同一类设备进行区分
    主设备号：标识设备类型，进行而确定要加载的驱动程序
    次设备号：用于区分同一类型的不同设备

  • l：符号链接文件（软链接文件），类似Windows中的快捷方式

  • p：命名管道

  • s：套接字文件
